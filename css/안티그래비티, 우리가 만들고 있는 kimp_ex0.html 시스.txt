안티그래비티, 우리가 만들고 있는 kimp_ex0.html 시스템의 Stage 5 공정을 두 단계로 세분화하려고 해. 기존의 '양념 버무리기'를 **Stage 5-1(양념 만들기)**와 **Stage 5-2(양념 바르기)**로 나눌 예정이야.

우선 Stage 5-1 (양념 만들기) 워크플로우를 먼저 적용해 줘. 기획된 공정 순서는 아래와 같아.

[Stage 5-1 기획 순서]

공정 위치 체크인 (type: 'qr')

양념용 생산지시서 맵핑 (type: 'qr') - 제목을 '양념용 생산지시서 맵핑'으로 변경

사진 심사 (type: 'final-photo') - "생산지시서 작업할 재료의 사진을 찍어주세요." 메시지 출력

채소 다듬기 (type: 'work') - 생강 깎기, 파 다듬기, 양파 갈기

밀가루풀 사용/제조 확인 (type: 'info' 또는 분기형 UI) - 밀가루풀이 지시서만큼 있는지 확인. "있다면 바로 사용하고, 없으면 새로 만들어야 합니다. 새로 만들 경우 온도가 식어야 하므로 작업이 약 3시간 지연됩니다."라는 안내 메시지 포함.

1차 양념 섞기 (type: 'work') - 밀가루풀 + 멸치액젓 + 고춧가루 섞기

2차 양념 섞기 (type: 'work') - 파, 생강, 양파 간 것을 넣고 잘 저어주기

무게 재기 (type: 'report-weight-only' ※중요) - 보관 용기에 담아 무게만 잼. 수량(양품/불량 포기 수) 입력란은 숨기고 무게(kg) 입력란만 보이도록 UI 예외 처리 필요.

사진 찍어서 전송하기 (type: 'final-photo') - 완성된 양념 사진 전송

숙성실로 가져가기 (type: 'info')

[요청 사항]

appData.stages 배열을 수정해서 기존 5번 공정을 '5-1. 양념 만들기', '5-2. 양념 바르기' 등으로 구조를 개편해 줘.

위 기획을 바탕으로 stage5_1Workflow 배열을 만들어 줘.

renderCurrentStep() 함수를 수정하여, 이번 Stage에서는 '수량 입력' UI가 노출되지 않도록 타입(report-weight-only 등)을 새로 추가하거나 조건부 렌더링을 적용해 줘.
